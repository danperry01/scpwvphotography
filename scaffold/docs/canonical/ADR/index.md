## What is an ADR?

An Architecture Decision Record captures an important architectural decision made along with its context and consequences.

## When to Write an ADR

- Choosing a technology or framework
- Defining a significant pattern or convention
- Making a decision with long-term implications
- Choosing between multiple valid approaches
- Changing an existing architectural pattern

## ADR Lifecycle

1. **Proposed** - Under discussion
2. **Accepted** - Decision made and implemented
3. **Deprecated** - No longer recommended
4. **Superseded** - Replaced by another ADR

## Naming Convention

ADRs are numbered sequentially: `adr-NNNN-short-title.md`

Examples:
- `adr-0001-use-cloudflare-workers.md`
- `adr-0002-database-selection.md`
- `adr-0003-authentication-approach.md`
